file double

file double

Jul 23, 20255 min read

  • Nom anglais: deque
  • Nom pleinement qualifié: std::deque

Une file double deque (double-ended queue) est un conteneur séquentiel faisant partie des conteneur standard de la STL en C++. Elle permet une insertion et une suppression rapides en tête comme en queue, contrairement à un vecteur std::vector qui est optimisé uniquement pour l’arrière.

Elle combine certains avantages du vecteur vector (accès rapide par index) et de list (insertion rapide en début), au prix d’une structure interne plus complexe.

Le nom deque est l’abréviation de double-ended queue, ce qui signifie en français : file à double extrémité

Comparaison C/C++ : Une deque est l’équivalent d’un tableau dynamique à deux extrémités, qui permet de croître ou décroître efficacement aux deux bouts.

Implémentation interne

Une deque<T> encapsule :

  • un tableau segmenté (tableaux discontinus appelés blocs ou buffers)
  • un système d’indexation pour accéder rapidement à chaque bloc
  • une mémoire non contiguë, mais un accès logique en séquence

Contrairement à un vecteur vector, les éléments d’une file double deque ne sont pas tous en mémoire contiguë, mais l’accès par index reste O(1) grâce à une structure d’index intermédiaire.

Deque et mémoire

La std::deque est conçue pour permettre des insertions et suppressions rapides aux deux extrémités. Pour cela, elle alloue la mémoire sous forme de blocs discontinus.

  • Accès par index : rapide (O(1)) mais légèrement plus lent que vector
  • Insertion/suppression en tête ou queue : très efficace (O(1) amorti)
  • Insertion au milieu : coûteuse (O(n))

Contrairement à std::vector (un vecteur alloue toute sa mémoire d’un seul tenant), une file std::deque fonctionne comme un tableau de pointeurs vers des blocs de données, où :

  • chaque bloc contient plusieurs éléments (souvent 512 ou 1024 octets selon l’implémentation)
  • les blocs ne sont pas nécessairement adjacents en mémoire
  • on peut ajouter un nouveau bloc à gauche ou à droite rapidement

Les performances en cache sont moindres par rapport à un vecteur mais meilleures qu’une liste.

Quand et pourquoi utiliser une file double

Une file (ou queue) est une structure adaptée aux situations où les données doivent être traitées dans l’ordre d’arrivée (FIFO – First In, First Out). Elle est particulièrement utile lorsqu’on doit gérer des tâches, des requêtes ou des événements dans l’ordre chronologique.

Par exemple :

  • une file d’attente de traitements
  • une file de messages réseau
  • ou une file de jobs dans un système multitâche.

La file garantit que le premier élément inséré sera aussi le premier traité, ce qui correspond à de nombreux modèles de communication ou de programmation orientée événements.

Remarques importantes

  • std::deque est idéale pour des accès par index combinés à des ajouts en tête
  • Elle remplace avantageusement une liste ou un vecteur (list ou vector) dans des cas spécifiques
  • Les insertions au milieu sont coûteuses (O(n)), comme pour vector
  • Les éléments ne sont pas contigus, donc la deque n’est pas cache-friendly

Utilisation comme base d’autres conteneurs

Le std::deque est non seulement un conteneur séquentiel à part entière, mais il sert aussi de conteneur sous-jacent par défaut pour plusieurs conteneur adaptateurs de la STL :

  • std::stack : pour une pile algorithmique LIFO
  • std::queue : pour une file FIFO

Cela s’explique par sa capacité unique à insérer et retirer efficacement aux deux extrémités, ce que ne permet pas un vecteur vector.
Grâce à cette flexibilité, deque est souvent préféré comme structure de support pour les structures d’accès restreint.

Il est également possible de substituer deque par d’autres conteneurs compatibles (comme vector ou list), mais deque reste le choix par défaut recommandé.
